Advantages of CQRS
1. Independent Scaling

Read traffic can be much higher than write traffic.

Example:

1 write/sec
10,000 reads/sec

Scale query servers separately.

2. Better Performance

Use:

SQL DB for writes
Redis / ElasticSearch for reads
3. Better Separation of Concerns

Write logic:

validation
transactions
business rules

Read logic:

caching
optimized projections

Drawbacks
1. Complexity

Instead of:

One Service

Now:

Command Service
Query Service
Event Bus
Read Model
2. Eventual Consistency

Read DB may lag behind write DB.

Example:

Order placed now
Read side updated after 1 second

User may not see data immediately.

3. More Infrastructure

Need:

Event Bus
Read DB
Synchronization
